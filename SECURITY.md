# Security policy

## Reporting a vulnerability

Do not open a public GitHub issue for a suspected vulnerability.

Email **security@getonramp.dev** with:

- The affected SDK, service, or URL
- Reproduction steps or a proof of concept
- Expected impact
- Any suggested mitigation

Please avoid accessing or changing other users' data while researching an issue. We will acknowledge a complete report as soon as practical and coordinate disclosure after a fix is available.

## Supported versions

Security fixes are applied to the latest published version of each OnRamp SDK. Upgrade to the newest release before reporting a problem that may already be resolved.
